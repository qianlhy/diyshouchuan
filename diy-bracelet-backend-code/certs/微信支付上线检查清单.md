## 二、你必须手动完成（否则无法真实支付）

### 1. 设置 APIv3 密钥（必须）

1. 登录 [微信支付商户平台](https://pay.weixin.qq.com/)
2. **账户中心 → API安全 → APIv3密钥 → 设置**
3. 设置 **32 位**随机字符串并保存（设置后无法查看）
4. 更新服务器配置：
   - 宝塔/Docker 环境变量：`SKY_WECHAT_APIV3KEY=你的32位密钥`
   - 或 `application.yml` 中 `sky.wechat.apiV3Key`

**当前状态：❌ 仍为占位值，必须替换**

### 2. 下载微信支付公钥（必须）

你下载的 `1113372390_20260531_cert` **只含商户 API 证书**，不含微信支付公钥。

1. 商户平台 → **账户中心 → API安全 → 微信支付公钥 → 申请公钥**
2. 下载公钥文件（如 `1113372390_wxp_pub.pem`）
3. 重命名并上传到服务器：`diy-bracelet-backend-code/certs/pub_key.pem`
4. 复制 **公钥ID**（格式 `PUB_KEY_ID_01xxxxxxxx`）
5. 更新配置：
   - `SKY_WECHAT_WECHATPAYPUBLICKEYID=新的公钥ID`

**当前状态：❌ pub_key.pem 仍是旧商户 1734348120 的公钥，必须更换**

## 五、常见报错

| 报错 | 原因 | 处理 |
|------|------|------|
| `ERR_CERT_AUTHORITY_INVALID` | 服务器 SSL 证书链不完整 | 宝塔重签 SSL |
| `SIGN_ERROR` / 401 | 证书序列号或私钥不匹配 | 核对 mchSerialNo 与 apiclient_cert.pem |
| `APPID_MCHID_NOT_MATCH` | 小程序未关联商户号 | 商户平台绑定 AppID |
| 响应签名验证失败 | pub_key.pem 或公钥ID 错误 | 重新下载新商户公钥 |
| 回调解密失败 | apiV3Key 错误 | 核对 32 位 APIv3 密钥 |
| 用户 openid 为空 | 未登录或登录失败 | 先解决 SSL，再重新登录 |
